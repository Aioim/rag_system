# RAG 优化策略全景图

> 以下按 RAG pipeline 的各个环节，系统性地梳理优化策略。

---

## 一、数据预处理 & 索引层

### 1. 文档解析与清洗
- **多格式解析**：PDF、Word、HTML、Markdown、图片(OCR)等统一解析
- **文档结构识别**：保留标题层级、表格、列表等结构信息
- **去噪清洗**：去除页眉页脚、水印、广告、无关字符
- **表格/图表专项处理**：表格转 Markdown/JSON 结构化存储，图表提取为文字描述
- **语义分块（Semantic Chunking）**：按语义边界切片，而非固定长度
- **滑动窗口分块**：相邻 chunk 有一定重叠，避免信息断裂
- **层级分块（Hierarchical Chunking）**：父子文档结构，粗粒度+细粒度并存
- **Small-to-Big 检索**：索引小 chunk，检索时召回大 chunk 喂给 LLM
- **Late Chunking / ColBERT**：先用长上下文 embedding，再在 token 级别做 late interaction 匹配——兼顾语义完整性和精确匹配
- **Contextual Retrieval（Anthropic）**：每个 chunk embedding 时前置整篇文档的上下文摘要，让向量包含文档级语境，解决孤岛 chunk 语义漂移
- **文档元数据提取**：标题、作者、日期、来源、分类标签、权限级别等

### 2. 多集合/多知识库分域
- **按领域分集合**：如 公司制度 / 技术文档 / 产品手册 / FAQ
- **按权限分级**：公开文档、部门内部、机密文档 — 不同访问控制
- **按语言分库**：中文文档集合、英文文档集合 — 跨语言检索时路由
- **按时效性分库**：长期知识库 vs 临时/新闻类知识库
- **动态路由**：查询时根据意图自动选择检索目标集合

### 3. 索引结构优化
- **多粒度索引**：文档级、段落级、句子级索引并存
- **关键词索引 + 向量索引双轨**：互补检索
- **稀疏向量索引（SPLADE / BM25）**：比 dense embedding 更适合精确关键词匹配
- **知识图谱索引（GraphRAG 基础）**：实体-关系-属性结构化存储，支持社区检测和多跳推理
- **摘要索引**：先生成每篇文档摘要，检索时先匹配摘要再下钻原文
- **多模态索引**：文本、图片描述、表格数据分别索引，统一检索

### 4. Embedding 模型管理
- **通用模型选型**：OpenAI text-embedding-3 / BGE / E5 / GTE 等按场景选型
- **领域微调（Fine-tuning）**：用领域标注数据微调 embedding 模型，提升领域语义对齐
- **Matryoshka Embedding**：支持可变维度截断，在检索效率和精度间灵活取舍
- **多语言 Embedding**：中英文对齐模型，支持跨语言检索
- **Embedding 质量评估**：MTEB 基准 + 领域内检索准确率定期评测

## 二、查询理解 & 改写层

### 5. 查询改写（Query Rewriting）
- **问题分解（子问题拆分 / Multi-hop Decomposition）**：复杂问题拆为多个子问题分别检索后合并
- **抽象→具体展开**：用户问"怎么提高效率"→改写为"时间管理方法+效率工具推荐"
- **具体→抽象泛化**：用户问太偏门的术语→泛化为更通用表达
- **多角度改写**：同一问题用不同表述生成多个 query → 多次检索合并
- **HyDE（Hypothetical Document Embedding）**：先生成假设答案→用答案做向量检索
- **Query2Doc**：生成伪文档后与原 query 拼接，丰富检索语义
- **Step-Back Prompting**：先抽象出更高层的问题再检索
- **查询扩展**：用同义词、相关词扩展原始 query
- **约束条件提取**：从 query 中提取筛选条件（时间范围、部门、文档类型等）→ 用于过滤检索结果

### 6. 意图识别与分类
- **意图分类器**：识别用户是想"了解概念"/"操作步骤"/"对比分析"/"查找具体数据"
- **问题清晰度判断**：识别模糊意图，触发对齐对话
- **领域分类**：自动识别问题属于哪个领域，路由到对应知识库
- **紧急程度判断**：是否需要即时回答 vs 可以异步处理
- **复杂度评估**：简单问题直接回答 vs 复杂问题触发多步检索

### 7. 多轮对话上下文管理
- **指代消解**："它"、"那个" → 替换为具体实体
- **对话历史压缩**：长对话只保留关键信息参与检索
- **话题切换检测**：检测到新话题时重置检索上下文
- **追问补全**：将追问补充为完整独立问题后再检索
- **上下文摘要注入**：对长对话生成摘要，摘要参与 query 改写

### 8. 用户画像与个性化
- **角色感知**：根据用户角色（新员工/管理者/技术岗）调整检索范围和回答深度
- **历史偏好学习**：根据用户历史行为调整排序权重
- **部门/组织上下文**：同部门文档优先，理解用户所在业务语境
- **知识水平适配**：新手给详细步骤，专家给要点和高阶内容

## 三、检索层（Retrieval）

### 9. 混合检索（Hybrid Search）
- **Dense + Sparse 融合**：语义向量 + BM25 关键词检索结果合并
- **RRF（Reciprocal Rank Fusion）**：排名融合算法
- **加权融合**：根据意图类型动态调整语义 vs 关键词权重
- **多路召回**：语义向量 / BM25 / 知识图谱 / 摘要索引等并行 → 结果融合去重
- **Late Interaction（ColBERT）**：token 级交互匹配，比单向量更精细，比 Cross-Encoder 更快

### 10. 检索策略优化
- **Reranking（重排序）**：粗召回后用 Cross-Encoder 精排
- **Cohere Rerank / BGE-Reranker / Cross-Encoder**：专用重排序模型
- **LLM-as-Reranker**：用 LLM 对候选文档打分排序
- **多样性控制**：MMR（最大边际相关性）算法避免冗余
- **时间衰减**：新文档加权，旧文档降权
- **权威性加权**：可信来源加权，非官方来源降权
- **上下文窗口感知检索**：根据 LLM 上下文窗口大小动态调整检索数量
- **Lost in the Middle 应对**：关键文档放首尾，中等相关度放中间，避免中间信息被 LLM 忽略

### 11. GraphRAG（知识图谱增强检索）
- **实体-关系抽取**：从文档中自动抽取实体及其关系构建知识图谱
- **社区检测（Leiden 算法）**：将图谱划分为语义社区
- **社区摘要生成**：每个社区生成结构化摘要，支持宏观问题回答
- **多跳推理检索**：沿图谱路径多步检索，回答"X 和 Y 有什么关系"类问题
- **Global + Local Search**：全局性问题用社区摘要，局部性问题用实体邻居检索
- **适用场景**：需要全局理解、跨文档关联、实体关系推理的问题

### 12. Self-RAG / CRAG（自适应检索增强）
- **Self-RAG**：
  - 按需检索：判断当前是否需要检索（有些问题 LLM 已知）
  - 检索相关性自评：检索后让模型评估"这些资料是否有用"
  - 多轮检索：资料不足时自动触发补充检索
  - 生成质量自评：生成答案后自评事实准确性、相关性
  - 选择性引用：只引用真正支撑答案的文档片段
- **CRAG（Corrective RAG）**：
  - 检索后先评估文档相关性评分
  - 评分高 → 直接生成；评分中 → 补充检索后生成；评分低 → 转联网搜索
  - 闭环纠错：生成答案后再验证，发现幻觉则重新检索

### 13. 迭代式检索
- **检索→评估→补充检索**循环：直到信息足够
- **查询细化迭代**：第一轮检索结果反馈给查询改写模块
- **逐层深入**：先摘要匹配→确定相关文档→深入全文检索
- **多跳检索链**：第一跳答案驱动第二跳检索，适用于需要多步推理的问题

### 14. Agentic RAG（智能体驱动检索）
- **检索 Agent 编排**：由 Agent 自主决定检索策略、工具选择、检索轮次
- **工具调用**：检索+计算器+数据库查询+API 调用等多工具组合
- **Plan-and-Execute**：先规划检索步骤 → 逐步执行 → 汇总生成
- **ReAct 模式**：思考→行动→观察→思考...的循环式检索推理
- **多 Agent 协作**：检索 Agent + 生成 Agent + 验证 Agent 分工协作
- **自主纠错**：检索结果矛盾或不足时自主调整策略

## 四、生成层（Generation）

### 15. Prompt 工程
- **动态领域/角色感知 Prompt**：根据问题领域自动注入对应的角色设定和提示词模板
- **Few-shot 示例动态选择**：根据 query 相似度检索最相关示例
- **思维链（Chain-of-Thought）**：引导逐步推理
- **结构化输出约束**：要求 JSON/Markdown 表格/特定格式
- **引用标注要求**：要求每个事实点标注来源和页码
- **System Prompt 分层设计**：全局约束 + 领域指令 + 当前场景补充，模块化组装

### 16. 上下文组装策略
- **文档顺序优化**：最相关文档放最前和最后（首尾效应），中等相关度放中间
- **Lost in the Middle 缓解**：关键信息前置，在 Prompt 中显式指示"请关注第一段和最后一段"
- **文档摘要注入**：检索结果太多时，先注入各文档摘要，节省窗口
- **上下文压缩**：检索文档过长时先用小模型压缩再送入生成模型
- **冗余去重**：相似片段去重，节省上下文窗口
- **信息密度最大化**：优先选择信息密度高的片段
- **多文档对比/冲突显式标注**：资料矛盾时明确指出"来源 A 说 X，来源 B 说 Y"，让用户知情

### 17. 生成控制
- **温度调节**：事实类问题低温度，创意类问题高温度
- **最大长度控制**：根据问题复杂度动态调整
- **拒答机制**：检索置信度过低时声明"资料不足以回答"，而非编造
- **不确定性标注**：对不确定内容加"可能""据现有资料"等限定词
- **引用溯源**：每个关键断言附带来源链接/文档标识
- **后验事实检查**：生成后由验证模型逐条核实事实与检索文档的一致性（Fact-checking）

### 18. 知识冲突解决
- **冲突检测**：识别检索文档中互相矛盾的陈述
- **多数投票**：多个来源一致优先
- **权威性仲裁**：高权威来源覆盖低权威来源
- **时效性仲裁**：新版本覆盖旧版本
- **透明呈现**：向用户列出不同来源的不同说法，而非自行裁决

## 五、用户交互 & 反馈层

### 19. 需求对齐对话
- **问题意图不清晰时的 Grill-me 式对齐**：主动提问澄清用户真实需求
- **多义提问分析**：问题有歧义时列出 "您是指...? A、B 还是 C?"
- **范围确认**："需要什么详细程度？概述还是详细步骤？"
- **预期管理**：告知回答覆盖范围 & 未覆盖部分

### 20. 别名映射 / 术语规范化
- **用户术语→标准术语映射表**：如用户说"工资条"映射到系统里的"薪资明细"
- **同义词词典**：缩写、俗称、别名 ↔ 正式名称的双向映射
- **动态别名学习**：从用户反馈中持续学习新的别名对应关系
- **多语言术语映射**：中英文术语互相映射
- **模糊匹配容错**：拼写错误、拼音输入、近似词自动纠正

### 21. 生成结果呈现
- **引用标注展示**：每条关键信息标注可点击的来源链接
- **置信度指示**：高/中/低置信度用视觉标识区分
- **追问建议**：生成答案后主动推荐 2-3 个相关追问
- **来源文档清单**：列出本次回答参考的全部文档
- **反馈入口**：👍/👎 + 可选文字反馈

### 22. 反馈与持续优化
- **用户反馈收集**：👍/👎 或打分，低分触发 Bad Case 入库
- **Bad Case 分析**：检索失败 case 的根因分析（检索不到 vs 检索到但生成错）
- **A/B 测试框架**：对比不同策略效果（不同 chunk size、不同 embedding 模型等）
- **检索日志与监控**：query、检索结果、生成质量指标仪表盘
- **知识库更新机制**：文档变更后的增量索引更新，过期文档自动下线
- **反馈驱动微调**：用标注的 Bad Case 数据微调 embedding 模型或 reranker
- **漂移检测**：监控检索命中率/生成质量指标趋势，及时发现性能退化

## 六、多模态 RAG

### 23. 多模态文档理解
- **图片理解**：文档中的截图、架构图、流程图 → 用视觉模型生成文字描述后索引
- **表格理解**：表格转结构化 JSON/Markdown，保留行列语义
- **图表理解**：柱状图/折线图/饼图 → 提取数据趋势和关键数值
- **音视频转写**：会议录音、培训视频 → ASR 转文字后索引
- **图文混合检索**：用户上传图片提问 → 图片理解 + 文字检索混合召回
- **多模态 Embedding（CLIP 等）**：图文统一向量空间检索

## 七、安全与护栏

### 24. 安全防护
- **Prompt 注入防御**：用户输入与检索文档分离，检索文档不参与系统指令解析
- **越权防护**：检索结果按用户权限过滤，禁止返回无权限文档内容
- **敏感信息脱敏**：手机号、身份证、薪资等 PII 在生成阶段自动脱敏
- **有害内容过滤**：输入/输出双端内容安全检测
- **知识库投毒防御**：文档上传时检测恶意注入内容

### 25. 护栏机制
- **话题边界控制**：超出知识库范围的问题触发兜底策略而非强行回答
- **角色越狱防护**：拒绝绕过系统设定的越狱尝试
- **合规审计日志**：记录所有 query、检索路径、生成结果的完整链路
- **熔断机制**：异常高频查询或攻击行为自动限流

---

## 八、降级 & 兜底策略

### 26. 资料不足时的兜底
- **网络搜索回退（Fallback to Web Search）**：本地知识库不足时自动触发网络搜索（需运行时支持）
- **联网搜索 + 本地文档融合**：两者结果整合后生成答案
- **声明式降级**：明确告诉用户"资料库中未找到相关内容，以下来自网络"
- **知识库范围告知**：让用户知道当前回答基于哪些知识源
- **人工转接**：关键业务场景下，AI 无法回答时自动创建人工工单

### 27. 性能 & 成本优化
- **查询缓存**：高频问题缓存答案（语义相似度匹配命中）
- **Embedding 缓存**：避免重复向量化
- **分层检索**：先查缓存 → 再查本地 → 最后联网
- **LLM 选择策略（模型路由）**：简单问题用小模型，复杂问题用大模型
- **延迟优化**：预加载热文档向量到内存，并行检索多集合
- **流式输出**：检索完成后逐 token 流式生成，降低用户感知延迟
- **成本追踪**：每次查询的 token 消耗和费用实时监控

## 九、评估体系

### 28. RAG 评估框架

| 维度 | 指标 | 说明 |
|------|------|------|
| 检索质量 | Recall@k, Precision@k, MRR, NDCG, Hit Rate, MAP | 衡量检索是否找到了相关文档 |
| 生成忠实度 | Faithfulness, Hallucination Rate | 生成内容是否忠于检索到的文档 |
| 答案相关性 | Answer Relevance, Context Relevance | 答案与问题的匹配度，检索上下文与问题的匹配度 |
| 上下文精度 | Context Precision, Context Recall | 检索的文档中有多少真正有用，有用文档的召回比例 |
| 噪声鲁棒性 | Noise Robustness | 检索到无关文档时生成质量是否下降 |
| 端到端框架 | RAGAS, ARES, TruLens, DeepEval, LangSmith | 自动化评估 pipeline |
| 用户体验 | 满意度、首响时间、解决率、LLM-as-Judge | 人工评估 + LLM 模拟评估 |

### 29. 评估方法
- **离线评估**：用标注的 (query, relevant_docs, ground_truth_answer) 数据集自动评分
- **合成数据生成**：LLM 自动从文档库生成问答对，构建评估集
- **在线评估**：生产环境的用户反馈 + A/B 实验
- **人工评估**：定期抽样人工评分，校准自动评估指标
- **回归测试集**：维护核心问答 benchmark，每次变更前跑回归

## 十、策略全景总览

```
用户 Query
    │
    ├── [安全护栏层]
    │   ├── Prompt 注入检测
    │   ├── 权限校验
    │   └── 内容安全过滤
    │
    ├── [查询理解层]
    │   ├── 意图分类 / 复杂度评估 → 领域/集合路由
    │   ├── 约束提取（时间/部门/类型过滤）
    │   ├── 查询改写：子问题拆分 / HyDE / Query2Doc / 多角度扩展
    │   ├── 别名映射 + 术语规范化
    │   ├── 用户画像注入（角色/偏好/知识水平）
    │   └── 意图不清晰？→ Grill-me 需求对齐
    │
    ├── [检索层]
    │   ├── 多路召回：语义向量 + BM25 + 知识图谱 + 摘要索引
    │   ├── 混合检索（Hybrid Search）+ RRF / 加权融合
    │   ├── Late Interaction（ColBERT）
    │   ├── GraphRAG：社区检测 → 多跳推理
    │   ├── Reranking（Cross-Encoder / LLM）
    │   └── Self-RAG / CRAG：相关性自评 → 按需补充检索或转联网搜索
    │
    ├── [Agentic 编排层]（可选）
    │   ├── ReAct：思考→检索→观察→再思考
    │   ├── 多工具调用（检索 + 计算 + API + 数据库）
    │   └── 多 Agent 协作（检索 Agent + 生成 Agent + 验证 Agent）
    │
    ├── [生成层]
    │   ├── 动态角色/领域感知 Prompt 组装
    │   ├── 上下文组装：首尾效应 / 摘要注入 / 冲突标注
    │   ├── Lost in the Middle 缓解
    │   ├── 引用溯源 + 事实核查
    │   └── 自信度评估 → 不确定性标注
    │
    ├── [后处理 & 兜底层]
    │   ├── 后验事实检查（Fact-checking）
    │   ├── 敏感信息脱敏
    │   ├── 资料充足？→ 直接输出
    │   ├── 资料不足？→ 网络搜索回退 → 融合输出
    │   └── 均无结果？→ 诚实告知 + 人工转接
    │
    ├── [呈现层]
    │   ├── 流式输出 + 引用标注
    │   ├── 置信度指示 + 追问推荐
    │   └── 反馈入口（👍/👎）
    │
    └── [反馈闭环]
        ├── 用户反馈 → Bad Case 入库分析
        ├── 别名映射持续学习更新
        ├── Embedding / Reranker 微调
        ├── 检索质量仪表盘 → 漂移检测
        └── 知识库增量更新 + 过期文档下线
```

## 十一、策略选型速查表

| 场景 | 推荐策略组合 |
|------|-------------|
| 知识库文档量 < 1K | Hybrid Search + Reranking + HyDE |
| 知识库文档量 1K-100K | + 多集合分域 + 摘要索引 + 查询改写 |
| 知识库文档量 > 100K | + GraphRAG + Agentic RAG + 模型路由 |
| 多轮对话场景 | + 指代消解 + 对话压缩 + 追问补全 |
| 需要全局理解（总结/对比/趋势） | GraphRAG 社区摘要 + 多跳检索 |
| 精确查找（制度/政策/数据） | BM25 + 别名映射 + 结构化输出约束 |
| 多语言场景 | 多语言 Embedding + 术语映射 + 语言分库 |
| 高安全要求（金融/医疗/政务） | 权限过滤 + 脱敏 + 审计日志 + 人工转接 |
| 实时性要求高 | 查询缓存 + 流式输出 + 预加载热向量 |
| 多模态文档（图文混排） | 多模态理解 + 图文混合检索 |
